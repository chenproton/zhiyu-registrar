

# 数字教务平台需求文档
## 系统能力清单（System Capability Manifest）

> **文档用途**：供项目团队、开发方及业务方理解本系统的完整能力边界、实体模型、操作规则及与周边子系统的集成关系。  
> **系统定位**：面向职业院校的数字教务管理与教学运行中枢平台，承担基础数据治理、教学运行调度、学业结果认定三大核心职能。  
> **版本**：v1.0.0  
> **最后更新**：2026-05

---

## 一、系统定位与边界

### 1.1 系统定位

数字教务平台是教学运行的**中枢底座**，向下对接数字课程服务平台、实践场景学习平台、能力测评认证平台，向上为师生提供统一教务视图。核心职能包括：

- **基础数据治理**：统一管理组织架构（院系-专业-班级）、师资、学生学籍、教学资源（教材/场地）等主数据，为各业务子系统提供唯一可信数据源
- **教学运行调度**：基于人才培养方案进行排班排课、调课、教学日历管理，汇聚教学进度数据，支撑教学质量评价
- **学业结果认定**：汇聚课程平台、场景平台、测评平台的学业数据，统一进行成绩认定、学分统计、学历认定

### 1.2 系统边界（明确不做/其他平台负责）

| 边界项 | 说明/负责方 |
|--------|-----------|
| 课程教学内容编辑 | **数字课程服务平台**负责（体系课/颗粒课/知识点/测验/录播视频） |
| 实践场景任务编辑 | **实践场景学习平台**负责（场景/任务链/评分/能力点映射/场景资源） |
| 题库/试卷/在线考试 | **能力测评认证平台**负责（组卷/考试/随堂测/题目管理） |
| 能力画像生成引擎 | **能力测评认证平台**负责（画像计算/岗位推荐/能力徽章） |
| 毕业设计全流程 | **能力测评认证平台**负责（选题/档案/评价/答辩） |
| 能力认证规则配置 | **能力测评认证平台**负责（岗位能力点拆解/等级映射/胜任力标准） |
| 学业预警引擎 | **本期不做**（二期规划功能） |
| AI辅助功能 | **本期不做**（如 AI 辅助编制培养方案） |
| 学生质疑/教学客诉 | **本期不做**（待定功能） |

### 1.3 数据汇聚边界

本平台**不存储原始学习过程数据**，仅通过标准接口汇聚结果数据：
- 从**课程平台**汇聚：课程节点完成率、测验成绩、学习进度百分比、出勤率
- 从**场景平台**汇聚：场景任务完成率、场景评分结果、学生实践进度
- 从**测评平台**汇聚：考试成绩、毕设合格状态、能力画像摘要、学历/能力徽章

## 三、核心实体与状态

### 3.1 组织架构（Organization）

| 实体 | 关键字段 | 说明 |
|------|---------|------|
| 院系（Department） | 院系编码、名称、类型、负责人、状态 | 支持启用/禁用 |
| 专业（Major） | 专业编码、名称、所属院系、培养层次、学制、状态 | 支持专业方向细分 |
| 班级（Class） | 班级编码、名称、所属专业、入学年份、班主任、学生数 | 行政班/教学班两种性质 |
| 学年/学期（Term） | 学年、学期、起止日期、当前学期标记 | 全局时间基准 |

**业务规则**：
- R-ORG01：院系禁用后，下属专业/班级不可新增，历史数据保留
- R-ORG02：班级学生数由学籍关联自动统计，不支持手动修改
- R-ORG03：专业变更学制时，仅影响后续入学年级，历史年级保留原学制

|------|------|
| 教师工号 | 唯一标识，对接统一身份认证 |
| 基础信息 | 姓名、性别、所属院系、所属专业、职称、学历 |
| 授课资格 | 可授课程分类标签、企业导师资格标记 |

| 状态 | 在职/离职/外聘 |

### 3.3 学生学籍管理（Student & StudentStatus）

学生基础信息与学籍状态统一维护，支持从入学到毕业的全周期学籍异动。

#### 学生基础信息

| 字段 | 说明 |
|------|------|
| 学号 | 唯一标识 |
| 基础信息 | 姓名、性别、身份证号、所属院系、专业、班级、入学年份 |
| 学籍状态 | 在籍/休学/退学/毕业/结业 |
| 学历档案摘要 | 培养方案版本、已修学分、GPA（由本平台成绩认定模块计算） |
| 能力档案袋摘要 | 关联能力测评平台的档案材料数量与类型统计（仅展示，不存储明细） |
| 学籍异动记录 | 转专业、休学、复学、留级等历史记录 |

#### 学籍状态流转

```
在籍(active) → 休学(suspended) → 在籍(active)
在籍(active) → 退学(dropped)
在籍(active) → 毕业(graduated) / 结业(completed)
```

**业务规则**：
- R-STU01：仅"在籍"学生可参与排课与成绩认定
- R-STU02：学生转专业时，已修课程成绩保留，学分按新专业培养方案重新认定
- R-STU03：学生档案中的能力档案袋数据从测评平台汇聚展示，本平台不做原始存储

### 3.4 人才培养方案（TrainingProgram）

| 字段 | 说明 |
|------|------|
| 方案名称 | 如"2026级软件工程专业人才培养方案" |
| 方案编码 | 系统自动生成 |
| 适用专业 | 关联专业与入学年级 |
| 培养层次 | 中专/大专/本科 |
| 学制 | 学年数 |
| 毕业学分要求 | 总学分、必修课学分、选修课学分、实践学分 |
| 课程计划 | 课程列表（课程编码/名称/学分/学时/学期/课程性质/考核方式） |
| 能力认证要求 | 关联能力测评平台的岗位能力认证规则ID（仅引用，不配置规则内容） |
| 版本号 | 系统自动管理 |
| 状态 | 草稿/审批中/已发布/已废止 |

**业务规则**：
- R-TP01：同一专业+入学年级仅允许一个已发布方案
- R-TP02：方案发布后方可用于该年级排课与学分认定
- R-TP03：方案废止后，在读学生仍按原方案执行，新生按新方案执行
- R-TP04：本期不做 AI 辅助编制，支持手动配置课程计划与上传方案文档

| 字段 | 说明 |
|------|------|

- R-SC02：课表发布前支持手动调整；发布后师生端可见
- R-SC03：场景教学类课程建议关联场景平台已发布场景，缺失时黄色警告

**业务规则**：

| 适用对象 | 课程类型/教师类型/教学环节 |

#### 评价活动（EvaluationActivity）

| 字段 | 说明 |

| 评价范围 | 按院系/专业/课程/教师筛选 |
| 参与人员 | 按角色自动匹配评价主体与被评价对象 |
| 活动时间 | 开始时间/结束时间 |
| 状态 | 未开始/进行中/已结束 |

#### 评价记录（EvaluationRecord）

| 字段 | 说明 |
|------|------|
| 关联活动 | 关联 EvaluationActivity |
| 评价主体 | 评价人（学生/教师/专家/企业导师） |
| 被评价对象 | 教师/课程/教学环节 |
| 评价维度得分 | 各维度评分 |
| 总评 | 综合得分/等级 |
| 评语 | 文本（可选） |

**业务规则**：
- R-TQE01：学生评教采用匿名方式，学生提交后方可查看自己的成绩（可配置强制关联）

- R-TQE03：企业评价仅针对有企业导师参与的实践课程或场景教学
- R-TQE04：评价结果按院系/专业/教师维度汇总排名，生成教学质量报告
- R-TQE05：同一评价活动中，同一评价主体对同一被评价对象仅可提交一次

### 3.10 成绩认定（GradeRecognition）

#### 成绩记录（GradeRecord）

| 字段 | 说明 |
|------|------|
| 学生 | 关联 Student |
| 课程/场景 | 课程平台课程ID 或 场景平台场景ID |
| 成绩类型 | 平时/期中/期末/实践/总评/补考/重修 |
| 原始成绩 | 从子系统同步的原始分数 |
| 认定成绩 | 经教务审核后的有效成绩 |
| 学分 | 该课程对应学分（从培养方案读取） |
| 绩点 | 根据认定成绩自动计算 |
| 认定状态 | 待确认/待审核/待认定/已认定/已发布 |

**成绩认定流程**：

```
子系统原始成绩同步 → 教师确认 → 院系审核 → 教务认定 → 发布
```

**业务规则**：
- R-GR01：成绩发布前学生不可见；发布后学生端可查看
- R-GR02：补考/重修成绩单独标记，覆盖或并行记录按学校政策配置
- R-GR03：总评成绩计算权重从人才培养方案课程计划的考核方式配置读取
- R-GR04：教师仅可确认本人授课班级的成绩；院系管理员审核本院系成绩；教务管理员最终认定与发布

### 3.11 学历认定（DegreeRecognition）

本期做简化版：仅做数据汇聚展示与最终审核发布，不做复杂自动校验引擎。

| 字段 | 说明 |
|------|------|
| 学生 | 关联 Student |
| 培养方案 | 关联 TrainingProgram |
| 学分统计 | 已获总学分/必修学分/选修学分/实践学分（由本平台成绩认定模块自动汇总） |
| 课程成绩合格状态 | 必修课合格数/总数（本平台统计） |
| 毕设状态 | 从能力测评认证平台同步（合格/不合格） |
| 出勤率 | 从课程平台/场景平台汇聚（仅展示原始数据） |
| 能力达成摘要 | 从能力测评平台同步（岗位胜任力达标状态） |
| 学历认定结果 | 待审核/符合毕业条件/不符合毕业条件/结业/肄业 |
| 学历徽章 | 从能力测评认证平台同步 |

**业务规则**：
- R-DR01：学历认定结果由教务管理员基于汇聚数据进行**人工审核确认**，系统提供辅助统计（学分汇总、不合格课程列表、缺失条件提示）

## 五、跨系统业务流程

### 流程1：学期教学运行初始化

```
教学日历配置（教务平台）
    ↓
人才培养方案确认（教务平台）→ 提取课程计划
    ↓
教学资源准备（教材/场地录入）（教务平台-数据治理中心）
    ↓

    ↓
课表同步至数字课程服务平台（学生端课表展示）
    ↓
场景教学课表关联实践场景学习平台场景ID（教务平台记录场景ID）
    ↓
教学进度监控开启（教务平台每日定时同步课程/场景进度数据，仅展示）
    ↓
教学质量评价活动配置（教务平台-评价模板+活动创建）
```

### 流程2：成绩认定与发布

```
课程平台产生学习数据/测验成绩
场景平台产生场景任务评分
测评平台产生考试/随堂测成绩
    ↓
教务平台定时同步原始成绩
    ↓
教师确认成绩（教务平台教师端）
    ↓
院系管理员审核成绩（教务平台）
    ↓
教务管理员认定并发布成绩
    ↓
学分自动统计（用于学历认定辅助展示）
```

### 流程3：毕业资格学历认定（简化版）

```
学生达到毕业年级
    ↓
教务平台自动汇聚：
  - 学分统计（本平台成绩认定模块）
  - 必修课合格状态（本平台统计）
  - 毕设合格状态（能力测评平台同步）
  - 出勤率（课程/场景平台汇聚展示）
  - 能力达成摘要（能力测评平台同步）
    ↓
系统提示缺失条件（辅助功能）
    ↓
教务管理员人工审核确认
    ↓
学历认定发布 → 学生端可查询毕业资格与学历徽章
```

### 流程4：教学质量评价闭环

```
教务管理员配置评价模板（学生/内部/专家/企业）
    ↓
创建评价活动，绑定学期与评价范围
    ↓
启动评价活动，通知各评价主体参与
    ↓
各主体参与评价：
  - 学生评教（学生端匿名提交）
  - 内部评价（教师互评/督导评价）
  - 专家评价（专家端听课记录与评分）
  - 企业评价（企业导师端提交）
    ↓
评价活动结束，锁定数据
    ↓
生成教学质量报告（按教师/课程/院系汇总排名）
    ↓
评价结果应用于教师教学质量档案
```
