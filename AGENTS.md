# Agent 工作规范

## 任务启动前的评估与确认

**在开始任何编码任务前，必须先评估用户提供的信息是否足够。**

如果信息不足，**主动提出需要补充的问题**，等我补充完信息后再开始任务。如果我补充的信息仍然不足，可以进行多轮提问，直到需求足够清晰为止。

### 为什么需要这个规范

以班级作息表为例，最初只收到：
> "不要再重复选择院系和班级了，直接配置当前院系和当前年级的所有班级作息方案。"

但实现后发现以下关键信息缺失，导致反复修改：
1. 作息表的展示形式是"列表"还是"表格"（横轴/纵轴）？
2. 数据是否需要和其他功能联动？
3. 是否需要保留"支持单双周"等配置项？
4. 每个单元格里显示什么内容？
5. 是独立配置还是批量应用到所有班级？

**因此，信息不充分时直接开始编码，往往会导致返工。**

### 执行标准

- 收到任务后，先判断需求是否完整
- 对模糊点、缺失点、有歧义的地方，立即列出问题清单
- 等用户确认/补充后，再进入编码阶段
- 复杂需求可先给出方案草图，用户确认后再实现
