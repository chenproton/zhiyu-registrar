# 数字教务平台 — 以学习任务为最小单元的架构设计文档

> **文档版本**：v2.0.0  
> **设计目标**：将"学习任务"作为教学运行的最小业务单元，重构教学运行中心，统一串联建课、备课、发布、执行、测评、进度、复盘全生命周期。  
> **适用范围**：教学运行中心（Teaching Operations Center）  

---

## 一、架构设计总览

### 1.1 核心设计理念

**旧模式**：教学运行以"课表条目（TimetableEntry）"为最小单元，排课、成绩、进度、评价各自独立，数据分散在不同模块中，没有统一的业务载体串联。

**新模式**：以"**学习任务（Learning Task）**"为教学运行的最小业务单元。一个任务实例串联起从建课到归档的完整教学闭环。

```
旧模式：           新模式（任务载体）：
┌─────────┐        ┌─────────────────────────────────────────┐
│  排课   │        │              学习任务实例                │
│ (时间)  │        │  ┌─────────┐ ┌─────────┐ ┌───────────┐ │
├─────────┤        │  │ 建课信息 │ │ 备课资源 │ │  发布状态  │ │
│  成绩   │   →→   │  ├─────────┤ ├─────────┤ ├───────────┤ │
│ (分数)  │        │  │ 课堂组织 │ │ 测评关联 │ │  进度数据  │ │
├─────────┤        │  ├─────────┤ ├─────────┤ ├───────────┤ │
│  进度   │        │  │ 课后复盘 │ │ 评价记录 │ │  归档状态  │ │
│ (完成率)│        │  └─────────┘ └─────────┘ └───────────┘ │
└─────────┘        └─────────────────────────────────────────┘
```

### 1.2 双视角模型

同一个任务实例，根据角色展示不同的视图和操作权限：

| 维度 | 教师端：教学任务（Teaching Task View） | 学生端：学习任务（Learning Task View） |
|------|--------------------------------------|--------------------------------------|
| **可见字段** | 全部字段（含备课资源、成绩明细、学生列表、企业导师安排） | 学生可见字段（课程名、时间地点、学习资源、个人成绩、个人进度） |
| **核心操作** | 备课、发布资源、布置子任务、确认成绩、查看班级学情、教学复盘 | 查看任务要求、访问学习资源、查看个人成绩/进度、参与评教 |
| **数据范围** | 整个班级/教学班的聚合数据 | 仅个人相关数据 |

### 1.3 任务生命周期

```
                    ┌─────────────┐
                    │   草稿 draft  │
                    └──────┬──────┘
                           │ 导入Excel 或 手动创建
                           ▼
                    ┌─────────────┐
                    │ 待就绪 ready │ ← 细分安排、资源准备、冲突检测
                    └──────┬──────┘
                           │ 发布
                           ▼
             ┌─────────────────────────────┐
             │        已发布 published        │
             │  （学生端可见学习任务）         │
             └─────────────┬───────────────┘
                           │ 到达上课时间
                           ▼
             ┌─────────────────────────────┐
             │      进行中 in_progress        │
             │  （课堂教学执行期）             │
             └─────────────┬───────────────┘
                           │ 课程结束，进入成绩评定
                           ▼
             ┌─────────────────────────────┐
             │      评定中 evaluating         │
             │  （成绩确认/审核/认定流程中）    │
             └─────────────┬───────────────┘
                           │ 成绩发布
                           ▼
             ┌─────────────────────────────┐
             │      已完成 completed          │
             │  （成绩已发布，可进入复盘）      │
             └─────────────┬───────────────┘
                           │ 复盘完成/学期结束
                           ▼
                    ┌─────────────┐
                    │  已归档 archived│
                    │  （进入学业档案）│
                    └─────────────┘
```

### 1.4 系统边界重申

| 能力 | 本系统负责 | 外部子系统负责 |
|------|-----------|--------------|
| 任务的时间/人员/场地编排 | ✓ | — |
| 任务状态流转与发布控制 | ✓ | — |
| 任务与外部资源的关联引用 | ✓ | — |
| 备课资源的上传与存储 | ✓ | — |
| 任务成绩汇聚与认定 | ✓ | — |
| 课程内容的录播/测验编辑 | — | 数字课程服务平台 |
| 场景任务链/评分规则配置 | — | 实践场景学习平台 |
| 题库/组卷/在线考试 | — | 能力测评认证平台 |
| 原始学习过程数据存储 | — | 各子系统 |

## 二、核心实体模型

### 2.1 任务主实体（Task）

> 这是新架构的核心表，替代原有的 `TimetableEntry` 成为教学运行的最小单元。

```typescript
interface Task {
  // === 基础标识 ===
  id: string                    // 任务唯一ID
  code: string                  // 任务编码，如 T-2026A-SE101-001
  name: string                  // 任务名称，如 "软件工程2026级1班-程序设计基础"
  
  // === 任务分类 ===
  type: 'traditional' | 'scene' // 传统教学 | 实践场景教学
  source: 'imported' | 'manual' // 来源：外部导入 | 手动创建
  status: TaskStatus            // 任务生命周期状态
  
  // === 关联基础数据 ===
  termId: string                // 学期ID
  courseName: string            // 课程名称（从培养方案引用或手动录入）
  courseCode?: string           // 课程编码
  
  // === 教学组织 ===
  classId: string               // 班级ID
  className: string             // 班级名称
  
  // === 教师端 ===
  facultyId: string             // 主讲教师ID
  facultyName: string           // 主讲教师名称
  enterpriseMentorId?: string   // 企业导师ID（场景教学必填）
  enterpriseMentorName?: string // 企业导师名称
  
  // === 时空安排 ===
  dayOfWeek: number             // 星期几 1-7
  period: string                // 节次，如 "1-2节"
  weeks: string                 // 周次范围，如 "1-16周"
  
  // === 场地资源 ===
  venueId: string               // 场地ID
  venueName: string             // 场地名称
  workStationId?: string        // 工位/设备ID（场景教学用）
  workStationName?: string      // 工位/设备名称
  
  // === 外部平台关联 ===
  externalPlatformId?: string   // 外部平台课程/场景ID
  externalPlatformType?: 'course' | 'scene' // 关联平台类型
  
  // === 任务内容（备课层）===
  resources: TaskResource[]     // 教学资源列表（教材、课件、链接等）
  syllabus?: string             // 教学大纲/任务说明
  objectives?: string[]         // 教学目标/任务目标
  
  // === 场景教学特有（细分安排层）===
  sceneSubTasks?: SceneSubTask[] // 场景教学细分课时任务
  
  // === 进度汇总（从外部平台同步）===
  progressSummary?: TaskProgressSummary
  
  // === 成绩汇总 ===
  gradeSummary?: TaskGradeSummary
  
  // === 复盘 ===
  review?: TaskReview
  
  // === 元数据 ===
  createdAt: string
  updatedAt: string
  publishedAt?: string          // 发布时间
  completedAt?: string          // 完成时间
  archivedAt?: string           // 归档时间
}

type TaskStatus = 
  | 'draft'        // 草稿（导入后初始状态）
  | 'ready'        // 待就绪（细分安排完成，待发布）
  | 'published'    // 已发布（学生端可见）
  | 'in_progress'  // 进行中（课堂教学执行期）
  | 'evaluating'   // 评定中（成绩确认/审核中）
  | 'completed'    // 已完成（成绩已发布）
  | 'archived'     // 已归档
```

### 2.2 任务资源（TaskResource）

> 承载"备课"和"发布资源"环节，教师上传的资源学生可见。

```typescript
interface TaskResource {
  id: string
  taskId: string
  name: string                    // 资源名称，如 "第1章课件.pdf"
  type: 'textbook' | 'ppt' | 'video' | 'link' | 'document' | 'scene_link'
  url?: string                    // 资源链接/路径
  textbookId?: string             // 关联教材库ID
  externalResourceId?: string     // 外部平台资源ID
  isVisibleToStudents: boolean   // 是否对学生可见
  uploadBy: string                // 上传者（教师ID）
  uploadedAt: string
  sortOrder: number               // 排序
}
```

### 2.3 场景教学子任务（SceneSubTask）

> 承载场景教学的"细分课时任务关联"。场景教学在场景平台已拆分子任务/任务链，教务侧需要将教师与这些细分任务做关联。

```typescript
interface SceneSubTask {
  id: string
  taskId: string                  // 所属主任务
  name: string                    // 子任务名称，如 "场景一：需求分析实战"
  scenePlatformTaskId: string     // 场景平台子任务ID
  
  // === 师资关联 ===
  facultyId?: string              // 负责该子任务的教师（可与主任务不同）
  facultyName?: string
  enterpriseMentorId?: string     // 企业导师
  enterpriseMentorName?: string
  mentorParticipationType: 'full' | 'partial' | 'remote' // 企业导师参与方式：全程/部分/远程
  mentorTimeSlots?: TimeSlot[]    // 企业导师到场时段
  
  // === 时空安排（可与主任务不同）===
  dayOfWeek?: number              // 如子任务需单独安排时间
  period?: string
  weeks?: string
  
  // === 设备/工位 ===
  workStationId?: string
  workStationName?: string
  equipmentList?: string[]        // 所需设备清单
  
  // === 状态 ===
  status: 'planned' | 'in_progress' | 'completed'
  
  // === 进度（从场景平台同步）===
  progress?: {
    completionRate: number        // 学生平均完成率
    syncAt: string                // 同步时间
  }
}

interface TimeSlot {
  date: string                    // 日期，如 "2026-09-15"
  period: string                  // 时段，如 "上午(1-4节)" / "下午(5-8节)"
}
```

### 2.4 任务进度汇总（TaskProgressSummary）

> 从外部平台汇聚的进度数据，挂载在任务上统一展示。

```typescript
interface TaskProgressSummary {
  // === 计划 vs 实际 ===
  plannedHours: number            // 计划课时
  completedHours: number          // 已完成课时
  completionRate: number          // 课时完成百分比
  
  // === 学生维度 ===
  studentAvgCompletion: number    // 学生平均学习完成率（课程平台/场景平台同步）
  studentCount: number            // 班级学生总数
  completedStudentCount: number  // 已完成学生数
  
  // === 子任务维度（仅场景教学）===
  subTaskProgress?: {
    subTaskId: string
    subTaskName: string
    completionRate: number
  }[]
  
  // === 同步元数据 ===
  dataSource: 'course_platform' | 'scene_platform' | 'mixed'
  lastSyncAt: string
}
```

### 2.5 任务成绩汇总（TaskGradeSummary）

> 一个任务下的所有成绩记录汇总，替代原有的分散成绩查询。

```typescript
interface TaskGradeSummary {
  taskId: string
  classId: string
  facultyId: string
  
  // === 成绩组成（按培养方案考核方式）===
  components: {
    type: 'usual' | 'midterm' | 'final' | 'practice' | 'total' | 'makeup' | 'retake'
    typeLabel: string              // 平时/期中/期末/实践/总评/补考/重修
    weight: number                 // 权重
    status: 'pending' | 'confirmed' | 'audited' | 'recognized' | 'published'
    recordCount: number            // 已录入记录数
    totalStudents: number         // 应录总数
  }[]
  
  // === 整体状态 ===
  overallStatus: 'pending' | 'evaluating' | 'published'
  
  // === 发布控制 ===
  publishAt?: string
  publishedBy?: string
}
```

### 2.6 课后复盘（TaskReview）

> 承载"课后复盘"环节，教师在任务完成后撰写教学反思。

```typescript
interface TaskReview {
  taskId: string
  
  // === 教师复盘 ===
  facultyReview?: {
    teachingReflection: string     // 教学反思
    problemsFound: string[]        // 发现问题
    improvementMeasures: string[]  // 改进措施
    studentFeedbackSummary: string // 学生反馈摘要
    createdAt: string
  }
  
  // === 学生复盘（可选，作为学习任务的一部分）===
  studentReviews?: {
    studentId: string
    learningSummary: string        // 学习总结
    difficulties: string[]         // 遇到的困难
    suggestions: string[]          // 对教学的建议
    createdAt: string
  }[]
  
  // === 评价结果快照 ===
  evaluationSnapshot?: {
    avgScore: number               // 平均评分
    participantCount: number       // 参与人数
    dimensionScores: Record<string, number>
  }
}
```

## 三、教学运行中心重构设计

### 3.1 整体模块架构

```
教学运行中心
├── 📅 教学日历（基本不变，任务时间属性与之关联）
│   └── 学期管理 / 周次配置 / 节假日标记
│
├── 📋 任务编排中心（原排课管理重构）
│   ├── 外部排课导入
│   ├── 任务初审与标记
│   ├── 场景教学细分安排
│   ├── 任务资源准备（备课）
│   ├── 冲突检测与发布
│   └── 任务列表/甘特图/课表视图
│
├── 🔄 任务调度（原调课管理重构）
│   ├── 任务变更申请（时间/地点/人员/资源）
│   ├── 变更审批
│   └── 变更通知与状态同步
│
├── 📊 任务进度看板（原教学进度监控重构）
│   ├── 任务维度进度总览
│   ├── 场景子任务进度明细
│   ├── 班级/学生进度穿透
│   └── 外部平台同步日志
│
├── ⭐ 任务评价（原教学质量评价重构）
│   ├── 评价模板
│   ├── 评价活动
│   └── 评价结果（按任务实例汇总）
│
└── 📝 任务复盘
    ├── 教师复盘撰写
    ├── 学生复盘收集（可选）
    └── 复盘归档与标杆提取
```

---

### 3.2 任务编排中心（详细需求）

> **定位**：替代原有的"排课管理"，成为教学运行的核心调度模块。不再从零排课，而是**以外部导入为起点，以任务细化和发布为终点**。

#### 3.2.1 外部排课导入

| 需求项 | 说明 |
|--------|------|
| **导入模板** | Excel 文件，字段：课程名、课程编码、班级、教师、星期、节次、周次、场地、**课程性质（理论/实践/场景教学）** |
| **导入方式** | 单文件上传，支持预览确认 |
| **冲突检测** | **不检测**（信任外部排课软件），仅做格式校验（必填字段、数值范围） |
| **导入结果** | 每一行生成一个 `Task` 实例，状态 = `draft`，source = `imported` |
| **课程性质识别** | 优先读取 Excel 的"课程性质"列；缺失时支持批量手动标记 |

#### 3.2.2 任务初审与标记

| 需求项 | 说明 |
|--------|------|
| **初审界面** | 导入后进入"待处理任务"列表，展示所有 draft 状态任务 |
| **批量标记** | 可批量选择任务，标记为"传统教学"或"场景教学" |
| **信息补全** | 补充缺失字段：课程编码、班级ID（关联系统班级）、教师ID（关联系统师资）、场地ID（关联系统场地） |
| **异常标记** | 未匹配到系统内班级/教师/场地的行标红提示，需手动关联或创建 |

#### 3.2.3 传统教学任务发布流程

```
draft → 补充外部平台课程ID（可选） → 上传备课资源 → ready → 发布 → published
```

| 需求项 | 说明 |
|--------|------|
| **关联课程平台** | 可选：关联数字课程服务平台的课程ID，用于后续进度同步 |
| **备课资源** | 教师可上传/关联：教材、课件PPT、视频、文档、外部链接 |
| **快速发布** | 传统教学任务信息补全后，可直接批量发布 |

#### 3.2.4 场景教学细分安排流程（核心差异化功能）

```
draft → 信息补全 → 关联场景平台 → 拆分/关联子任务 → 分配工位设备 → 安排企业导师时段 → ready → 发布 → published
```

| 步骤 | 功能需求 | 字段/操作 |
|------|---------|----------|
| **① 关联场景** | 绑定实践场景学习平台的场景ID | `externalPlatformId`、`externalPlatformType='scene'`；未关联时黄色警告 |
| **② 子任务关联** | 从场景平台同步该场景下的子任务/任务链列表，教务侧做师资关联 | `sceneSubTasks[].scenePlatformTaskId`、`facultyId`、`enterpriseMentorId` |
| **③ 工位/设备分配** | 为场景教学分配具体的实训工位或设备 | `workStationId`、`equipmentList` |
| **④ 企业导师时段** | 安排企业导师的参与方式（全程/部分/远程）和具体到场时段 | `mentorParticipationType`、`mentorTimeSlots[]` |
| **⑤ 子任务时空微调** | 如子任务需脱离主任务时间单独安排，可配置独立时间 | `dayOfWeek`、`period`、`weeks`（可选，默认继承主任务） |
| **⑥ 完成检查** | 检查必填项：场景ID、子任务师资关联、企业导师（如标记为企业导师参与型场景） | 缺失项红色提示，补全后方可流转至 ready |

#### 3.2.5 任务发布

| 需求项 | 说明 |
|--------|------|
| **发布条件** | 状态 = ready，必填字段完整 |
| **发布动作** | 点击"发布"后，状态变更为 `published`，同时生成学生端的 `StudentTaskView` |
| **发布时间** | 可设置定时发布，或立即发布 |
| **发布通知** | 发布后可选择是否推送通知给相关师生 |
| **发布后限制** | 已发布任务修改时间/地点需走"任务调度（调课）"流程 |

#### 3.2.6 多维视图

| 视图 | 用途 |
|------|------|
| **任务列表** | 以任务为行，展示状态、类型、时间、教师、班级、进度等关键字段，支持筛选和批量操作 |
| **课表视图** | 保留原有班级/教师/场地课表，但底层数据来源于 Task 实例 |
| **甘特图视图** | 按时间线展示任务的执行周期，场景教学可展开显示子任务 |
| **待办看板** | 按状态（draft / ready / published / in_progress）展示任务卡片 |

### 3.3 任务调度（原调课管理重构）

> **定位**：不再只是改时间地点，而是对**任务实例**的整体变更调度。

#### 3.3.1 变更类型

| 变更类型 | 适用对象 | 说明 |
|---------|---------|------|
| **时间变更** | 全部 | 修改星期/节次/周次 |
| **场地变更** | 全部 | 修改 venueId |
| **教师变更** | 全部 | 修改 facultyId |
| **企业导师变更** | 场景教学 | 修改 enterpriseMentorId 或时段 |
| **工位/设备变更** | 场景教学 | 修改 workStationId / equipmentList |
| **子任务调整** | 场景教学 | 增删改子任务的师资/时间 |
| **停课** | 全部 | 标记任务暂停，需配置补课计划 |
| **补课** | 全部 | 创建补课任务实例 |

#### 3.3.2 变更流程

```
申请人（教师/教务）发起变更 → 填写变更原因 → 变更影响预览（展示受影响的师生）
    ↓
审批人（院系管理员/教务管理员）审批
    ↓
通过：自动更新任务状态，生成变更记录，推送通知给相关师生
驳回：申请人收到驳回原因，任务保持原状
```

#### 3.3.3 变更记录

- 所有变更自动记录到 `TaskChangeLog`，包含：变更前/后快照、变更人、审批人、变更时间
- 师生端查看任务时，可看到"变更历史"

### 3.4 任务进度看板（原教学进度监控重构）

> **定位**：从"课程维度"升级为"任务维度"，支持穿透到子任务和学生个体。

#### 3.4.1 数据同步

| 来源 | 同步内容 | 同步周期 |
|------|---------|---------|
| 数字课程服务平台 | 课程节点完成率、测验成绩、学习进度百分比、出勤率 | 每日定时 |
| 实践场景学习平台 | 场景任务完成率、场景评分结果、子任务进度 | 每日定时 |

#### 3.4.2 看板层级

```
全局看板（教务管理员）
    └── 按院系/专业/班级筛选
            └── 任务卡片（展示：计划课时/已完成/完成率/学生平均完成率）
                    └── 点击进入任务详情
                            ├── 传统教学：展示学生列表及个人进度
                            └── 场景教学：展示子任务列表及各子任务进度
                                    └── 子任务穿透：展示学生个体在该子任务上的完成情况
```

#### 3.4.3 关键指标

| 指标 | 计算方式 |
|------|---------|
| 任务完成率 | 已完成课时 / 计划课时 |
| 学生平均完成率 | 学生个体完成率的平均值（从外部平台同步） |
| 子任务完成率 | 场景教学专用，各子任务的独立完成率 |
| 同步延迟 | 上次同步时间与当前时间的差值，超期标红 |

### 3.5 任务评价（原教学质量评价重构）

> **定位**：评价对象从"教师/课程"抽象维度，下沉到"任务实例"维度，实现更精准的教学质量追踪。

#### 3.5.1 评价维度扩展

在原有评价维度（教学态度/教学内容/教学方法/教学效果）基础上，新增任务相关维度：

| 新增维度 | 适用评价主体 | 说明 |
|---------|------------|------|
| 任务设计合理性 | 学生评教 | 学习目标是否清晰、资源是否充足 |
| 场景教学组织 | 学生评教/企业评价 | 子任务安排是否合理、企业导师指导是否到位 |
| 资源准备充分性 | 学生评教 | 教材/课件/场景资源是否充分 |
| 进度把控 | 内部评价/专家评价 | 任务进度是否合理 |

#### 3.5.2 评价关联粒度

- **评价活动**可按任务类型筛选（仅评价传统教学 / 仅评价场景教学 / 全部）
- **评价记录**关联到具体的 `taskId`，而不仅是 `facultyId`
- 同一教师的不同任务，可独立评价

#### 3.5.3 评价与复盘的联动

- 评价活动结束后，评价结果自动写入 `TaskReview.evaluationSnapshot`
- 教师撰写复盘时，可引用评价数据作为参考

---

### 3.6 任务复盘（新增模块）

> **定位**：承载"课后复盘"环节，形成教学改进的闭环。

#### 3.6.1 教师复盘

| 功能 | 说明 |
|------|------|
| **触发条件** | 任务状态 = completed（成绩已发布） |
| **撰写界面** | 表单：教学反思（富文本）、发现问题（多行文本列表）、改进措施（多行文本列表）、学生反馈摘要 |
| **数据辅助** | 自动带入：任务进度数据、评价结果快照、成绩分布统计 |
| **提交** | 提交后复盘内容关联到任务，进入归档流程 |

#### 3.6.2 学生复盘（可选）

| 功能 | 说明 |
|------|------|
| **触发条件** | 任务状态 = completed，且教师配置了"收集学生复盘" |
| **收集方式** | 学生端填写：学习总结、遇到的困难、对教学的建议 |
| **用途** | 教师复盘时可查看汇总数据 |

#### 3.6.3 复盘归档

- 复盘完成后，任务状态可由 `completed` 流转至 `archived`
- 已归档任务进入学业档案，不可再修改
- 优秀复盘可提取为教学成果申报素材

## 四、跨系统数据流

### 4.1 任务编排期的数据流

```
【外部排课软件】导出 Excel（课程/时间/教师/场地）
           ↓
【数字教务平台】导入 → 生成 Task（status=draft, source=imported）
           ↓
    ┌──────┴──────┐
    ▼             ▼
传统教学任务    场景教学任务
    │             │
    ▼             ▼
关联课程平台ID  关联场景平台ID
上传备课资源    同步子任务列表
                关联子任务师资
                分配工位设备
                安排企业导师时段
    │             │
    └──────┬──────┘
           ▼
   状态 → ready → 发布 → published
           ↓
【数字课程服务平台】接收课表数据
【实践场景学习平台】接收场景教学任务关联
```

### 4.2 任务执行期的数据流

```
【数字课程服务平台】
    产生：课程节点完成率、测验成绩、学习进度、出勤率
           ↓ API 每日同步
    【数字教务平台】更新 Task.progressSummary

【实践场景学习平台】
    产生：场景任务完成率、场景评分结果、子任务进度
           ↓ API 每日同步
    【数字教务平台】更新 Task.sceneSubTasks[].progress + Task.progressSummary
```

### 4.3 任务评价与成绩期的数据流

```
【数字教务平台】Task 进入 evaluating 状态
           ↓
    教师确认成绩 → 院系审核 → 教务认定
           ↓
    成绩发布 → Task.gradeSummary 更新 → Task 进入 completed
           ↓
    触发：评价活动（如配置为"成绩发布后评教"）
           ↓
    各主体评价 → 评价结果写入 TaskReview.evaluationSnapshot
           ↓
    教师撰写复盘 → Task 进入 archived
```

## 五、关键业务规则

| 规则ID | 规则描述 | 所属模块 |
|--------|---------|---------|
| R-TASK-01 | 任务编码全局唯一，格式建议：`T-{班级码}-{课程码}-{序号}` | 任务编排 |
| R-TASK-02 | 外部导入的任务初始状态 = draft，需完成细分安排后方可流转至 ready | 任务编排 |
| R-TASK-03 | 场景教学任务未关联 externalPlatformId 时，禁止流转至 ready（黄色警告） | 任务编排 |
| R-TASK-04 | 场景教学任务如标记需要企业导师，则必须配置 enterpriseMentorId 和 mentorTimeSlots | 任务编排 |
| R-TASK-05 | 已发布（published）任务修改时间/地点/教师，必须走任务调度（调课）流程 | 任务调度 |
| R-TASK-06 | 调课审批通过后，原任务状态不变，生成变更记录并更新任务字段 | 任务调度 |
| R-TASK-07 | 停课申请必须配置补课计划（关联补课任务ID），否则红色警告 | 任务调度 |
| R-TASK-08 | 教学进度数据从外部平台同步，本平台仅做汇聚展示，不做滞后预警 | 进度看板 |
| R-TASK-09 | 成绩认定以 Task 为粒度，一个 Task 下的所有成绩记录统一走确认→审核→认定→发布流程 | 成绩认定 |
| R-TASK-10 | 评价活动可精确到 taskId，同一教师的不同任务可独立评价 | 任务评价 |
| R-TASK-11 | 任务复盘仅在 completed 状态下可撰写，撰写后可选流转至 archived | 任务复盘 |
| R-TASK-12 | 已归档（archived）任务不可修改，仅可查看，数据进入学业档案 | 任务复盘 |
| R-TASK-13 | 学生端仅能看到 status ≥ published 的任务，且仅能查看个人相关数据 | 双视角 |
